# Security policy

## Supported version

Security fixes are applied to the current `main` branch and the latest tagged
release.

## Reporting a vulnerability

Use GitHub's private vulnerability-reporting or Security Advisory interface when
available. If a private channel is not available, open a minimal issue requesting
one without including exploit details, credentials, sensitive biological content,
or private data.

Relevant reports include fail-open routing or certification behavior, unsafe input
handling, credential exposure in the optional provider path, and release-artifact
integrity problems.

The package is research software and is not a substitute for laboratory, clinical,
or biosafety review.
